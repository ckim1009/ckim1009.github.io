# 3주차 이론 과제: RAG의 구성 요소

## 개요
이번 과제에서는 RAG(Retrieval-Augmented Generation)가 무엇이고, 어떤 구성 요소로 이루어져 있는지를 조사합니다. 실습에서 LangChain 기반 RAG 파이프라인을 직접 구현하기 위한 사전 지식을 갖추는 것이 목적입니다.

### 1. RAG란?
- LLM에 질의하기 이전에 외부 지식에서 검색을 시도한 뒤, 검색결과를 입력 프롬프트에 포함시켜 답변을 생성하는 기술입니다.
- RAG를 활용하면, 모델이 학습한 지식에서만 추론할 때 발생하는 할루시네이션 현상을 줄이고, 특정 도메인에 대한 학습 없이 올바른 대답을 유도할 수 있습니다.

#### 2주차 방법(system prompt에 전체 데이터 삽입)과 RAG의 차이
- System Prompt에 문서 데이터 전체를 삽입할 경우 컨텍스트 길이가 불필요하게 길어지는 문제 발생
- RAG의 경우 유사도 검색을 통해 문서에서 가장 적합한 부분만 프롬프트에 참조하여 컨텍스트의 낭비를 줄임

#### RAG 파이프라인의 전체 흐름
1. Indexing: 문서를 청킹한 뒤 임베딩하여 벡터 형태로 저장
2. Retrieval: 사용자 질문을 임베딩하여 유사도 검색을 통해 관련된 청크를 검색 및 추출
3. Generation: 검색한 청크와 사용자 질문을 결합해 프롬프트 구성하여 LLM에 질문

---

### 2. RAG 용어 정리
#### Chunking
- 긴 문서를 작은 단위(Chunk)로 나누는 과정
- `chunk_size`: 문서를 나누는 청크의 크기를 의미합니다.
- `chunk_overlap`: 문서를 나눌 때 청크의 앞 청크의 끝 부분와 뒷 청크의 시작부분을 겹치도록 만드는 설정입니다.

#### Embedding
- 텍스트를 벡터로 변환하는 방법.
- 주요 임베딩 모델
    - Sparse Embedding: 단어의 존재유무를 1,0 형식으로 표현하여 단어 개수만큼의 차원으로 벡터를 표현 (e.g. One-hot Encoding)
    - Dense Embedding: 텍스트를 저차원의 벡터로 전체 의미를 표현함. 벡터간의 연산으로 의미적 관계를 알 수 있음 (e.g. Word2Vec)

#### Vector Store
- 임베딩된 벡터를 저장하고 유사도 검색하는 저장소. FAISS, Chroma 

#### Retriever
- 질문과 유사한 청크를 검색하는 컴포넌트.
- `Top-K`: 유사도가 높은 상위 `K`개의 청크만 가져와 프롬프트에 제공

#### Generation
- 사용자 질문과 검색된 청크로 LLM에게 질문을 던져 답변을 생성하는 단계

---

### 3. 외부 데이터 소스 기반 응답 생성
#### LLM 단독 응답 vs 외부 데이터를 검색하여 응답하는 방식의 차이
- LLM 단독으로 응답할 시 모델이 학습한 내용만을 기반으로 대답하며, LLM의 특성 상 학습하지 않은 데이터일 경우 할루시네이션을 출력하는 문제가 발생할 수 있습니다.
- 외부 데이터를 검색하여 프롬프트에 포함할 시, 모델의 input에 정답에 해당하는 데이터가 첨부되어 있으므로, output을 출력할 때 이를 참조하여 정답을 답변할 수 있습니다. 

#### API 호출 흐름
1. 사용자 질문을 임베딩하여 벡터 생성

2. 벡터 스토어에서 유사 벡터(Top-K) 검색

3. 검색된 텍스트들로 시스템 프롬프트를 완성

4. 시스템 프롬프트와 사용자 프롬프트로 LLM API 호출

#### 할루시네이션 방지에서 RAG가 하는 역할
RAG는 LLM에게 답변의 근거를 직접 제공합니다. 모델 내 학습 데이터에 없는 내용을 지어내는 할루시네이션을 줄일 수 있습니다.

---

### 4. LangChain 기반 RAG 파이프라인 구조

단계	LangChain 컴포넌트
문서 로딩	Document Loader (PyPDFLoader 등)
청킹	Text Splitter (RecursiveCharacterTextSplitter 등)
임베딩	Embeddings (OpenAIEmbeddings 등)
벡터 저장	Vector Store (FAISS, Chroma 등)
검색 + 생성	Retriever + Chain

## 실습과제예측

### 의료급여 PDF를 청킹할 때 가장 큰 어려움이 무엇일지 예측
1. pdf에서 본문과 관련없는 페이지 구성요소들의 경우 제거가 필요하나 단순 청킹만으로는 제거가 불가능하여 검색에 방해가 될 것으로 예상
2. 표나 이미지의 경우 텍스트로 추출할 경우 구조가 깨지므로 RAG 활용 시 정확도가 떨어질 것으로 예상

### Golden Dataset의 5문제 중 어떤 난이도의 질문에서 검색 실패가 많을지 예측하고 이유를 설명
- 모든 난이도에서 검색 실패가 일어날 것으로 예상
- 청크안에 불필요한 데이터가 많이 포함되어 검색 효율이 떨어질 것으로 예상
