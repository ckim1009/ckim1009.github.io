# 2주차 과제: 프롬프트 엔지니어링


## 개요
LLM 모델이 의료급여 본인부담률 질문에 답하도록 대표적인 프롬프트 기법 4가지를 사용

# 프롬프트 기법 4가지 비교

### Zero-shot Prompting
- 정의: 프롬프트에 별도의 예시를 첨부하지 않고 원하는 결과를 유도하는 기법
- 원리: 모델이 주어진 문맥만으로 답을 파악
- 사용예시: 
```
질문
```
`답`

- 장점: 단순 응답, 정확도가 떨어지더라도 즉각적인 응답이 필요할 경우 유리
- 단점: 정확도가 중요하며, 복잡하고 예외적인 케이스가 많은 추론의 경우 불리

### Few-shot Prompting
- 정의: 원하는 출력과 관련된 입력-출력 예시를 하나 이상 프롬프트에 첨부하여 예시대로 출력하도록 유도하는 기법
- 원리: 프롬프트 내에 제공된 예시의 패턴을 기반으로 유사한 답을 제공
- 사용예시:

```
질문1
답1

질문2
답2

질문
```

`답`

- 장점: Zero-shot과 비교하여 예외적인 케이스도 추론 정확도를 높일 수 있음
- 단점: 예시 개수가 많아질수록 토큰 비용이 증가

### Chain-of-Thought (CoT)
- 정의: 모델의 추론과정을 단계별로 출력하여 원하는 결과가 나오도록 유도하는 기법
- 원리: 복잡한 문제를 작은 단위로 쪼개어 논리적인 흐름을 생성하게 함으로써, 추론 과정에서의 오류를 줄이고 정답 확률을 높임
- 사용예시:

```
질문1
중간과정1
중간과정2
중간과정3
답1

질문
```

`답`

- 장점: 수학적 문제, 복잡한 논리 문제에서 성능이 향상 
- 단점: 중간 과정이 길어지면 답변 생성이 느려지고 토큰 비용이 발생하며, 중간 단계에서 논리가 꼬이면 엉뚱한 결론에 도달

### Self-Consistency
- 정의: 모델이 여러 번 추론하게 한 뒤, 가장 많이 나온 답을 최종 결과로 선택하는 기법
- 원리: CoT를 기반으로 여러 개의 경로를 생성하고, 결과값들 중 다수결로 선택
- 장점: 단일 CoT에서 발생할 수 있는 논리 오류를 보정
- 단점: 동일 질문에 대해 컴퓨팅 자원과 비용이 가장 많이 소모

# 실습 과제 예측

1. Zero-shot에서 Easy, normal은 높게 나오나 hard부터는 정확도가 떨어질 것으로 예상
2. Easy, Medium은 크게 변동이 없으나 Hard에서 가장 큰 개선이 있을 것으로 예상
3. Zero-shot → Few-shot에서 가장 큰 개선이 있을 것으로 예상

### Step 2. Few-shot Prompting

```
#당신은 의료급여 본인부담률 질문에 정확히 답해야합니다.
#아래에 정리한 의료급여 정보만을 참조하여 답해야합니다.

{md_content}

## 예시
Q: 2종 수급권자의 조현병 외 정신질환 환자가 외래 진료를 받으면 본인부담률은 얼마인가요?

}

## 예시
Q: 2종 수급권자인 조현병 환자가 병원에서 외래 진료를 받았고 진료비가 100,000원입니다. 본인부담금은 얼마인가요?
A:
{
  "answer": "5,000원",
  "reason": "의료급여 2종 수급권자 조현병 환자의 외래 진료 본인부담률은 병원급 이상 5%입니다. 따라서 100,000원의 5%는 5,000원입니다."
}
```

### Step 3. Chain-of-Thought (CoT)
```
#당신은 의료급여 본인부담률 질문에 정확히 답해야합니다.
#아래에 정리한 의료급여 정보만을 참조하여 답해야합니다.

{md_content}

# 지시사항
1. 질문을 받으면 [추론 과정]과 [최종 답변]으로 나누어 작성하세요.
2. [추론 과정]에서는 다음 단계를 반드시 포함해야 합니다.
   - 대상자의 수급권자 구분 (1종/2종)
   - 질환 종류 및 진료 형태 (입원/외래, 조현병 여부 등)
   - 의료기관 종별 (의원/병원/상급종합병원 등)
   - 해당 조건에 맞는 본인부담률(%) 확인 및 산출 근거 제시
3. 최종 답변은 이전과 동일하게 JSON 형식으로 출력하세요.

## 예시
Q: 2종 수급권자인 조현병 환자가 병원에서 외래 진료를 받았고 진료비가 100,000원입니다. 본인부담금은 얼마인가요?

A:
[추론 과정]
1. 수급권자 확인: 2종 수급권자입니다.
2. 질환 확인: 조현병 환자입니다.
3. 진료 형태: 외래 진료이며, 의료기관은 병원급입니다.
4. 규정 확인: 제공된 정보에 따르면 2종 수급권자 중 조현병 환자의 외래 진료 본인부담률은 병원급 이상에서 5%입니다.
5. 계산: 진료비 100,000원 * 0.05(5%) = 5,000원입니다.

{
  "answer": "5,000원",
  "reason": "2종 수급권자 조현병 환자의 병원급 외래 본인부담률 5%를 적용하여 계산했습니다."
}
```


## 3. 결과 요약 테이블 -> 프롬프트 기반 별 구성

| 기법 | 프롬프트 구성 | 정확도 | 평균 총 토큰 |
| --- | --- | --- | --- |
| Zero-shot | 역할부여 + 참조 데이터 + 구조화 출력 | 30/30 | 1232.33  |
| Few-shot | 역할부여 + 참조 데이터 + 구조화 출력 + 예시 2개 | 30/30 | 1477.20 |
| CoT | 역할부여 + 참조 데이터 + 구조화 출력 + CoT | 30/30 | 1440.14  |
| Self-Consistency | 역할부여 + 참조 데이터 + 구조화 출력 + 3개의 CoT path | 30/30 | 1634.27 |


## 4. Step 1 → Step 4까지 정답률 개선 과정과 인사이트

- 예상과 다르게 모든 결과에서 정확도가 `1.0`으로 측정됨
- 사용한 모델의 성능이 좋아 결과도 좋게 나온 것으로 추측됨
- 토큰 사용량 측면에서 비교하였을 때, Zero-shot의 효율이 가장 높음을 확인
